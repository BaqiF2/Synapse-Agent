# Evaluation Rubric Details

Detailed guidance for scoring each evaluation criterion.

## Clarity Rubric

### Score 5 - Crystal Clear
- Description leaves no doubt about when to use the skill
- Every instruction is unambiguous
- Examples illuminate rather than confuse
- Technical terms are explained or obviously understood
- Navigation through the skill is intuitive

### Score 4 - Clear
- Description is good but could be more specific
- Most instructions are clear
- Examples are helpful
- Minor terminology issues
- Structure is logical

### Score 3 - Generally Clear
- Description is vague in some aspects
- Some instructions require interpretation
- Examples partially helpful
- Some undefined terms
- Structure has minor issues

### Score 2 - Frequently Unclear
- Description doesn't clearly specify triggers
- Many instructions are ambiguous
- Examples confusing or missing
- Terminology inconsistent
- Structure hard to follow

### Score 1 - Confusing
- Description is misleading or very vague
- Instructions are contradictory or unclear
- No useful examples
- Terminology used incorrectly
- No clear structure

## Completeness Rubric

### Score 5 - Comprehensive
- All common use cases covered
- Edge cases addressed
- Prerequisites clearly stated
- Error handling documented
- All referenced resources exist
- No gaps in workflows

### Score 4 - Mostly Complete
- Main use cases covered
- Most edge cases addressed
- Prerequisites stated
- Basic error handling
- References mostly complete
- Minor workflow gaps

### Score 3 - Adequate
- Core use cases covered
- Limited edge case coverage
- Some prerequisites mentioned
- Limited error handling
- Some missing references
- Noticeable workflow gaps

### Score 2 - Incomplete
- Not all main cases covered
- Edge cases ignored
- Prerequisites unclear
- No error handling
- Missing key references
- Major workflow gaps

### Score 1 - Missing Critical Content
- Core functionality not documented
- Would fail on basic use cases
- No guidance on prerequisites
- No error handling at all
- References broken or missing
- Workflows broken

## Usability Rubric

### Score 5 - Excellent
- Perfect flow from start to finish
- Logical progression
- Easy to find information
- Progressive disclosure works perfectly
- Best practices well integrated
- Could be used by someone unfamiliar with domain

### Score 4 - Good
- Good flow with minor friction
- Generally logical
- Information findable
- Progressive disclosure mostly works
- Best practices present
- Usable with basic domain knowledge

### Score 3 - Usable
- Workflow present but awkward
- Some logical issues
- Information scattered
- Progressive disclosure partially effective
- Some best practices
- Requires domain knowledge

### Score 2 - Difficult
- Workflow confusing
- Poor organization
- Hard to find information
- Progressive disclosure broken
- Few best practices
- Requires significant effort to use

### Score 1 - Unusable
- No clear workflow
- Chaotic organization
- Information buried or missing
- No progressive disclosure
- No best practices
- Would need complete rewrite

## Accuracy Rubric

### Score 5 - Verified Correct
- All technical information accurate
- Examples tested and working
- Scripts run without errors
- References up to date
- Commands/code snippets verified

### Score 4 - Correct with Minor Issues
- Technical information mostly accurate
- Examples work with minor adjustments
- Scripts work but may need tweaks
- Most references current
- Minor command/code issues

### Score 3 - Mostly Correct
- Some technical errors
- Examples work sometimes
- Scripts have bugs
- Some outdated references
- Multiple command/code issues

### Score 2 - Significant Errors
- Major technical errors
- Examples don't work
- Scripts fail
- Many outdated references
- Commands/code broken

### Score 1 - Fundamentally Wrong
- Core technical approach is wrong
- Examples are incorrect
- Scripts don't run at all
- References invalid
- Would lead to wrong outcomes

## Efficiency Rubric

### Score 5 - Optimal
- SKILL.md under 300 lines
- No unnecessary content
- Progressive disclosure fully utilized
- References well-organized
- Every word serves a purpose

### Score 4 - Efficient
- SKILL.md under 500 lines
- Minimal unnecessary content
- Good use of progressive disclosure
- References organized
- Little bloat

### Score 3 - Some Bloat
- SKILL.md 500-700 lines
- Some unnecessary content
- Progressive disclosure underutilized
- References could be better organized
- Noticeable bloat

### Score 2 - Bloated
- SKILL.md 700+ lines
- Significant unnecessary content
- Progressive disclosure not used
- References disorganized
- Major bloat

### Score 1 - Wasteful
- SKILL.md extremely long
- Massive duplication
- No progressive disclosure
- References chaotic
- Needs complete restructure
