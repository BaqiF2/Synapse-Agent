# Bug Fixer

## 快速开始
系统化修复代码BUG的技能，提供完整的识别→分析→修复→验证流程。

## 核心工作流程

### Phase 1: BUG识别与信息收集
```
1. 收集错误信息
   - 堆栈跟踪 (stack trace)
   - 日志文件 (logs)
   - 错误截图/输出
2. 获取重现步骤
   - 操作步骤序列
   - 环境信息 (OS, 版本, 配置)
   - 输入数据样本
3. 定位相关代码
   - 错误发生位置
   - 相关模块/文件
   - 依赖关系
```

### Phase 2: 根因分析
```
1. 重现BUG
   - 创建最小化重现案例
   - 验证重现一致性
2. 调试分析
   - 设置断点/日志
   - 分析执行路径
   - 检查变量状态
3. 确定根因
   - 数据流问题
   - 逻辑错误
   - 配置错误
   - 环境问题
```

### Phase 3: 修复方案设计
```
1. 选择修复策略
   - 热修复 (hot fix)
   - 重构 (refactor)
   - 配置调整 (config)
   - 依赖更新 (dependency)
2. 评估影响
   - 风险分析
   - 兼容性检查
   - 副作用评估
3. 制定测试计划
   - 单元测试
   - 集成测试
   - 回归测试
```

### Phase 4: 实施与验证
```
1. 编写修复代码
   - 最小化变更
   - 遵循编码规范
   - 添加注释说明
2. 测试验证
   - 运行测试套件
   - 手动验证流程
   - 性能影响检查
3. 部署上线
   - 代码审查
   - 渐进式发布
   - 监控观察
4. 文档更新
   - 更新变更日志
   - 完善文档
   - 知识分享
```

## 常见BUG类型参考
- **数据验证错误**: 参考 VALIDATION.md
- **并发问题**: 参考 CONCURRENCY.md
- **性能问题**: 参考 PERFORMANCE.md
- **内存泄漏**: 参考 MEMORY.md
- **API错误**: 参考 API_ERRORS.md

## 调试工具集成
- **断点调试**: VS Code, Chrome DevTools, GDB
- **日志分析**: grep, tail, ELK stack
- **性能分析**: Chrome Performance, perf, XCode Instruments
- **内存分析**: Valgrind, AddressSanitizer, Heap profilers

## 输出要求
```
修复报告应包含：
- BUG描述和影响
- 根因分析
- 修复方案
- 测试验证结果
- 预防措施建议
```
