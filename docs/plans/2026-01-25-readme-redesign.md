## 目标
将 README.md 从开发进度文档转变为面向开源项目使用者的产品文档。

## 用户需求分析

### 目标受众
开源项目使用者（开发者）

### 关键决策
1. **迁移进度**: 完全移除
2. **项目成熟度**: 标注为开发中/早期阶段
3. **核心特性**: 突出统一 Bash 抽象和扩展性设计

## 设计方案：极简实用型

### 设计原则
- 用户能在 2-3 分钟内理解项目并开始使用
- 突出核心概念（统一 Bash 抽象）
- 快速开始指南 + 基础示例
- 避免信息过载

### 文档结构

#### 1. 项目头部
- 项目名称和简短描述
- ⚠️ 开发状态警告
- 核心理念说明："一切工具都是 Bash"
- 特性列表（4 条）

#### 2. 快速开始
- 安装步骤（克隆、安装依赖、构建）
- 配置说明（.env 文件）
- 基础使用命令
- 简单示例展示 Agent 执行流程

#### 3. 架构概览
- 三层 Bash 工具体系图示
- Layer 1: Base Bash（原生命令）
- Layer 2: Agent Bash（核心工具）
- Layer 3: Field Bash（领域工具）
- 数据流图示

#### 4. 开发指南
- 测试命令
- 代码质量工具
- 项目结构说明
- 扩展开发示例（添加工具、添加技能）

#### 5. 文档链接与收尾
- 外部文档链接
- 技术栈列表
- 路线图（包含已完成和计划中的功能）
- 贡献说明
- 许可证
