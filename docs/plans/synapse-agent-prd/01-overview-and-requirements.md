# 第一部分：项目概述与需求

## 1. 项目概述

### 1.1 背景
当前 AI Agent 开发面临两大核心挑战：工具集成的碎片化和缺乏持续学习能力。现有 Agent 框架要么依赖固定的工具集，要么需要复杂的集成工作。Synapse Agent（神经突触）试图通过统一的抽象层和自我成长机制来解决这些问题。

### 1.2 核心理念

Synapse Agent 基于以下理念体系构建：

**已知定律**（基础假设）：
- **工具是 Agent 的手脚** - 工具赋予 Agent 行动能力
- **文件系统是记忆载体** - 持久化知识和经验的基础设施
- **技能的本质** - 技能由文件系统中的知识与可调用的工具组成

**待验证的核心假设**：
- **统一抽象层** - 一切工具都可以通过 Bash 接口统一调用
- **功能即工具** - Agent 的所有功能扩展都通过工具封装实现
- **智能的涌现** - 技能与 LLM 结合产生自我学习和自我成长能力
- **通用架构公式** - 技能 + LLM + Bash 构成通用智能体的完整形态
- **沙盒的定位** - 作为工程实现的工作台

### 1.3 项目目标
- **理念验证**：通过可运行的原型证明核心假设的可行性
- **自我成长**：实现 Agent 通过任务执行自动扩展技能库、强化已有技能的能力
- **通用框架**：构建可扩展的 Agent 框架原型，为未来多种交互方式（GUI、API等）奠定基础

---

## 2. 目标用户

### 2.1 当前阶段（验证期）
- **AI Agent 研究者** - 探索 Agent 自主学习和成长机制
- **开发者/技术爱好者** - 对通用 Agent 架构感兴趣，愿意通过 CLI 进行实验和验证
- **Agent 框架开发者** - 需要验证新的架构理念和实现方案

### 2.2 长期愿景
Synapse Agent 的理念面向所有 Agent 用户。当前 CLI 交互只是验证阶段的选择，框架设计考虑未来支持：
- GUI 图形界面
- API 服务
- 嵌入式集成
- 多模态交互

---

## 3. 核心使用场景

### 3.1 场景一：技能从零到有
**典型流程**：
1. 用户通过 CLI 输入任务："帮我分析这个 Python 项目的代码质量"
2. Agent 发现没有相关技能，通过调用工具和 Bash 完成任务
3. 任务完成后，技能强化 Agent 自动生成"代码质量分析"技能，保存到文件系统
4. 下次遇到类似任务时，直接加载该技能，提升执行效率

**价值验证**：证明"自我成长"能力 - 技能库从无到有的自动扩展

### 3.2 场景二：技能持续强化
**典型流程**：
1. 用户多次执行类似任务（如数据处理、代码重构等）
2. 技能强化 Agent 观察执行模式，优化技能中的工具调用顺序和参数
3. 技能质量指标（成功率、执行时间）持续提升
4. Agent 将优化后的工具组合封装为新的 Field Bash 命令

**价值验证**：证明"知识强化"能力 - 围绕技能+Bash的持续优化

### 3.3 场景三：外源技能导入与融合
**典型流程**：
1. 用户导入外部技能库（如 Claude Code Skills、自定义技能文档）
2. 技能搜索 Agent 分析外源技能的适用场景和工具依赖
3. Agent 将外源技能与已有技能进行对比，识别：
   - 需要强化的已有技能（学习外源技能的优化方案）
   - 需要新增的技能（填补能力空白）
4. 技能强化 Agent 自动完成技能融合，生成优化版本并保存

**价值验证**：证明"人类强化"能力 - Agent 可以从外部知识中学习和成长

### 3.4 场景四：工具转换为 Bash
**典型流程**：
1. **触发时机**：
   - 用户要求安装新工具时自动触发（如："帮我安装 git-tools MCP"）
   - 用户手动安装完 MCP/工具后，启动 Agent 时自动检测并运行转换脚本
   - 技能强化 Agent 新增工具时
   - 外源技能导入包含新工具时
2. 工具转 Bash 转换器自动识别工具来源并转换：
   - MCP 工具 → 调用 Mcp2Bash 转换器（启动时解析并缓存）
   - Skill 脚本 → 调用 Skill2Bash 转换器（后台监听进程自动生成）
3. 生成对应的 Field Bash 命令（mcp:* 和 skill:* 格式），安装到 PATH
4. Agent 可通过 `tools search` 发现工具，通过 `-h/--help` 获取使用说明

**价值验证**：证明"一切工具都是Bash"的可行性 - 统一抽象层的核心实现
